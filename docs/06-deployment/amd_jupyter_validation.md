# AMD Jupyter Validation Runbook

This runbook is the submission gate for the AMD notebook image **ROCm 7.2 + vLLM 0.16.0 + PyTorch 2.9**. Run every cell in order, then restart the kernel and repeat the complete run. Preserve the executed notebook, terminal logs, telemetry, results, and package inventory as submission evidence.

## 1. Prove the runtime

```python
import platform, subprocess, torch, vllm

print("python:", platform.python_version())
print("torch:", torch.__version__)
print("HIP:", torch.version.hip)
print("vLLM:", vllm.__version__)
print("GPU available:", torch.cuda.is_available())
assert torch.cuda.is_available()
print("GPU:", torch.cuda.get_device_name(0))
subprocess.run(["rocminfo"], check=True)
```

The recorded output must show PyTorch 2.9, ROCm/HIP 7.2, vLLM 0.16.0, and an AMD GPU. Record VRAM before loading the model with `rocm-smi --showmeminfo vram`.

## 2. Install and verify TERA dependencies

From the repository root:

```bash
python -m pip install -r backend/requirements.txt
PYTHONPATH=backend python -c "import lmdb, onnxruntime, tokenizers, jsonschema; print('TERA runtime imports OK')"
```

Do not reinstall `torch` or `vllm` from the TERA requirements file. The notebook image owns those ROCm-compatible packages.

The semantic cache requires a MiniLM ONNX model and a sibling `tokenizer.json`. Provision both files in a persistent notebook directory; they are not generated by TERA. Verify them before starting the application:

```bash
test -s /workspace/models/minilm.onnx
test -s /workspace/models/tokenizer.json
```

## 3. Start vLLM on port 8000

Use a model that fits the notebook GPU. This 7B model is the reference fast local tier:

```bash
mkdir -p /workspace/logs
vllm serve Qwen/Qwen2.5-7B-Instruct \
  --host 127.0.0.1 \
  --port 8000 \
  --dtype auto \
  --gpu-memory-utilization 0.85 \
  --max-model-len 8192 \
  > /workspace/logs/vllm.log 2>&1 &
```

Wait for readiness instead of using a fixed sleep:

```bash
until curl -fsS http://127.0.0.1:8000/health >/dev/null; do sleep 2; done
curl -fsS http://127.0.0.1:8000/v1/models
```

TERA uses port 7860. Do not start TERA on port 8000 because that port belongs to vLLM.

## 4. Validate the OpenAI-compatible response and logprobs

```python
import requests

payload = {
    "model": "Qwen/Qwen2.5-7B-Instruct",
    "messages": [{"role": "user", "content": "Return only the number: 2 + 2"}],
    "temperature": 0,
    "max_tokens": 16,
    "logprobs": True,
    "top_logprobs": 1,
}
r = requests.post(
    "http://127.0.0.1:8000/v1/chat/completions", json=payload, timeout=120
)
r.raise_for_status()
body = r.json()
assert body["choices"][0]["message"]["content"]
assert body["usage"]["total_tokens"] > 0
assert body["choices"][0]["logprobs"]["content"], body
body
```

Nonempty token logprobs are mandatory because ROVL uses them for entropy and surprisal checks. An empty list is not acceptable validation evidence.

Start TERA on its separate port:

```bash

until curl -fsS http://127.0.0.1:7860/ready >/dev/null; do sleep 2; done
```

## 6. Exercise every route

Run at least:

1. One arithmetic, logic, and text-counting deterministic solver request.
2. One repeated prompt that proves an exact semantic-cache hit.
3. One ordinary prompt that reaches local vLLM and returns nonempty logprobs.
4. One schema-constrained and one regex-constrained request.
5. One deliberately forced local failure (stop vLLM or use an unreachable local port).

The forced local failure is a hard safety test: TERA must fail locally or use an explicitly configured second local tier. It must never attempt Fireworks. Search both logs for remote dispatch or Fireworks URLs after the test.

## 7. Run tests and the audited benchmark

```bash
python -m pytest -q
python scripts/run_benchmark.py
python scripts/analyze_benchmark.py
python scripts/audit_benchmark.py
```

Use the repository's actual benchmark arguments if its CLI requires explicit paths. Capture exit codes and preserve generated reports. Report accuracy, route distribution, cache/solver hit rates, locally generated tokens, latency p50/p95, and peak VRAM. The 99% accuracy goal must be computed over the full frozen evaluation set, not only successful or deterministic routes.

## 8. Assert zero Fireworks use

Run this against the produced telemetry before accepting the notebook run:

```python
import json
from pathlib import Path

p = Path("/workspace/output/telemetry.json")
raw = p.read_text(encoding="utf-8").strip()
try:
    parsed = json.loads(raw)
    rows = parsed if isinstance(parsed, list) else [parsed]
except json.JSONDecodeError:
    rows = [json.loads(line) for line in raw.splitlines() if line.strip()]

assert rows, "telemetry is empty"
assert all(int(r.get("fireworks_tokens", 0) or 0) == 0 for r in rows)
assert all(not r.get("remote_fallback_triggered", False) for r in rows)
assert all(r.get("route_taken") != "remote_fallback" for r in rows)
print(f"PASS: {len(rows)} records, zero Fireworks tokens and zero remote routes")
```

Also require both searches to return no matches attributable to the run:

```bash
grep -Ein "api\.fireworks\.ai|routing request to remote|remote_fallback" /workspace/logs/tera.log || true
grep -Ein "api\.fireworks\.ai" /workspace/logs/vllm.log || true
```

## 9. Clean-kernel reproducibility gate

Stop both servers, restart the Jupyter kernel, clear only transient process state, and rerun sections 1–8 without relying on variables from the first run. Do not delete the persistent model files or first-run evidence. A submission-ready run must satisfy all of the following twice:

- Exact target runtime and AMD device are recorded.
- TERA and vLLM become healthy on ports 7860 and 8000 respectively.
- Chat logprobs are present and parsed.
- The complete test suite passes.
- Benchmark accuracy meets the declared target on the frozen set.
- Telemetry contains zero Fireworks tokens, zero remote-fallback flags, and zero remote routes.
- Forced local failure produces no external request.
- Package inventory, GPU/VRAM output, server logs, telemetry, results, and the executed notebook are retained.
